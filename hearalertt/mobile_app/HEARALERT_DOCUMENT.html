1. Title

"HearAlert: AI-Powered Real-Time Sound Recognition and Multi-Modal Alert System for Deaf and Hard-of-Hearing Individuals"

2. Abstract

Hearing impairment significantly impacts an individual's ability to perceive environmental sounds critical for safety, communication, and daily living. According to the World Health Organization, over 430 million people worldwide require rehabilitation to address their hearing loss. Traditional assistive devices like hearing aids provide limited functionality and fail to address the diverse acoustic environments deaf individuals encounter.

This project presents HearAlert, an innovative mobile application leveraging Artificial Intelligence and Machine Learning to provide real-time environmental sound recognition and intelligent alerting specifically designed for deaf and hard-of-hearing users.

The system employs TensorFlow Lite's YAMNet model, a deep neural network trained on the AudioSet dataset containing over 2 million audio clips. This model is capable of classifying more than 500 distinct sound categories including:

  • Emergency Sounds: Fire alarms, sirens, smoke detectors, car alarms
  • Human Interactions: Doorbell, door knocking, telephone ringing, speech
  • Infant Sounds: Baby crying, infant distress signals
  • Animal Sounds: Dog barking, cat meowing, bird calls
  • Potential Dangers: Glass breaking, vehicle horns, gunshots, explosions

The application processes audio input at a 16kHz sampling rate with inference latency under 50 milliseconds, enabling near-instantaneous detection and response.

HearAlert implements a sophisticated multi-modal alert mechanism that delivers notifications through three primary channels:

  1. Haptic Feedback: Category-specific vibration patterns designed for intuitive recognition (e.g., SOS pattern for fire alarms, gentle pulses for baby cries)
  2. Visual Alerts: Flashlight strobe patterns synchronized with sound urgency levels
  3. Text-to-Speech: Voice announcements for users with partial hearing

A priority-based detection engine with 38+ prioritized sound categories ensures critical sounds receive immediate attention. The specialized Baby Cry Classifier module provides categorization of infant distress signals (hunger, discomfort, tiredness).

Keywords: Deaf Accessibility, Sound Recognition, Machine Learning, TensorFlow Lite, YAMNet, Flutter, Haptic Feedback, Mobile Application, Assistive Technology

3. Plan of Work

The development of HearAlert is divided into six distinct phases, each with specific objectives, tasks, and deliverables.

3.1 Phase 1: Research & Planning

Task Description
Literature Survey Study existing accessibility apps and research audio ML models
Requirement Analysis Gather functional and non-functional requirements
Tech Stack Selection Compare frameworks and select ML runtime
System Design Architecture design, database design, and UI wireframes

Key Activities:

  • Study existing apps: Sound Alert, Visualfy, Braci for feature comparison
  • Research YAMNet, VGGish, and AudioSet for audio classification
  • Survey deaf user needs through accessibility forums and WHO guidelines
  • Compare Flutter vs React Native vs Native development approaches
  • Design layered architecture with Provider state management pattern

3.2 Phase 2: Core Development

Task Description
Project Setup Flutter project initialization and platform configuration
Audio Capture Microphone stream implementation at 16kHz sampling rate
ML Integration TensorFlow Lite and YAMNet model integration
Priority Database Create database of 38+ prioritized sounds with confidence boosting

Key Activities:

  • Initialize Flutter project with Android, iOS, and Linux platform support
  • Configure microphone permissions and runtime permission handling
  • Implement real-time audio streaming using mic_stream package at 16kHz
  • Integrate TensorFlow Lite runtime with tflite_flutter plugin
  • Load and configure YAMNet model for 521-class audio classification
  • Implement audio preprocessing: normalization, buffering, and windowing
  • Create priority sound database with confidence boosting algorithms

3.3 Phase 3: Alert System

Task Description
Vibration Patterns Design category-specific patterns with millisecond specifications
Flashlight Alerts Torch API integration and strobe pattern implementation
TTS Integration Text-to-speech setup and announcement templates
Multi-Modal Sync Synchronized alert delivery across all modalities

Vibration Pattern Specifications:

  • Fire Alarm: SOS pattern – [100,100]×3 + [300,100]×3 + [100,100]×3 ms
  • Vehicle Horn: Long pulses – [500, 200, 500, 200, 500] ms
  • Door Knock: Double tap – [100, 50, 100, 300, 100, 50, 100] ms
  • Baby Cry: Gentle pulse – [200, 100, 200, 100, 200] ms
  • Glass Breaking: Urgent jitter – [50, 30]×10 ms
  • Doorbell: Chime pattern – [200, 100, 200, 300] ms

Flashlight Alert Patterns:

  • Emergency Alerts: Rapid strobe (100ms on/off) – fire, explosion, glass breaking
  • Warning Alerts: Medium strobe (300ms on/off) – vehicle horn, human distress
  • Informational Alerts: Slow flash (500ms on/off) – doorbell, phone ring

3.4 Phase 4: UI/UX Development

Task Description
Liquid Glass Theme Color palette, typography, and glassmorphism effects
Screen Implementation Home, Live Detection, Settings, and History screens
Accessibility Features High contrast mode, large text, deaf card, signal guide

Screen Implementations:

  • Home Screen: Dashboard with quick actions, detection status, recent alerts
  • Live Detection: Real-time waveform visualization and classification results
  • Settings Screen: Vibration intensity, theme selection, notification preferences
  • History Screen: Timeline view of past detected sounds with filtering
  • Signal Guide: Interactive guide for learning vibration pattern meanings
  • Contacts Screen: Emergency contact management for SOS feature

Accessibility Features:

  • High Contrast Mode: Enhanced color contrast for visibility
  • Large Text Support: Scalable typography for readability
  • "I'm Deaf" Card: Digital communication card for interactions
  • Screen Flash: Full-screen color flash for critical alerts

3.5 Phase 5: Advanced Features

Task Description
Baby Cry Classifier Custom ML model for infant cry categorization
Scenario Profiles Home, Street, and School detection modes
Emergency SOS One-tap emergency trigger with contact alerts

Baby Cry Categories:

  • Hungry: Rhythmic, repetitive cry – Try feeding the baby
  • Needs Burping: Fussy cry – Gently pat baby's back
  • Belly Pain: High-pitched, intense cry – Check for colic, tummy massage
  • Discomfort: Whimpering cry – Check diaper, clothing, temperature
  • Tired: Breathy, yawning cry – Create calm environment, try rocking

Scenario Profiles:

  • Home Mode: Prioritizes doorbell, baby cry, appliances, smoke alarm
  • Street Mode: Prioritizes vehicle horns, sirens, traffic sounds
  • School Mode: Prioritizes bells, announcements, fire alarms
